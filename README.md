## Descripción del sistema

Este proyecto consiste en un sistema básico de gestión de restaurante desarrollado en Python utilizando Programación Orientada a Objetos (POO). El sistema permite registrar productos disponibles en el restaurante, registrar clientes y administrar esta información mediante una clase principal llamada Restaurante.

El objetivo es demostrar la organización de un proyecto en módulos, la separación de responsabilidades entre clases y el uso de importaciones entre archivos para construir una aplicación estructurada y fácil de mantener.

## Descripción de los archivos

- *modelos/producto.py:* contiene la clase Producto, que representa los productos disponibles en el restaurante.
- *modelos/cliente.py:* contiene la clase Cliente, que representa a los clientes registrados.
- *servicios/restaurante.py:* contiene la clase Restaurante, encargada de gestionar los productos y clientes.
- *main.py:* archivo principal donde se crean los objetos y se ejecutan las funcionalidades del sistema.

## Reflexión

La modularización del software es importante porque permite organizar el código en diferentes archivos según su función, facilitando su comprensión y mantenimiento. Además, la separación de responsabilidades ayuda a que cada clase tenga una tarea específica, haciendo que el programa sea más ordenado, reutilizable y fácil de ampliar en el futuro. Aplicar estos principios mejora la calidad del desarrollo y permite trabajar de manera más eficiente en proyectos de cualquier tamaño.
